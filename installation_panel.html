Обзор

Административная панель для управления установкой оборудования

Продукт — внутренний инструмент для отслеживания процесса монтажа умных камер и устройств на транспортные средства. Менеджеры контролируют статус установок в разрезе компаний-клиентов, типов транспорта и команд установщиков.

Задача

Три сущности — один интерфейс

Данные об установках были разрозненны: группы инсталляций, команды установщиков и транспортные средства существовали по отдельности. Задача — собрать три взаимосвязанных модуля в единый портал.

просматривать группы установок с детализацией по отделам и статусам;
управлять реестром установщиков: список, профиль, создание;
видеть привязанные транспортные средства и их типы;
провалиться из любой сущности в связанную за один клик.
Три сущности, три таблицы в разных местах — задача собрать их так, чтобы менеджер не терял контекст при переходе между ними.
Решения

5 экранов — сквозная навигация по трём модулям

Каждый модуль самодостаточен, но связан с остальными: из группы установок — в детали установки и установщика, из установщика — в его транспортные средства.

01
Группы установок

Сводный список всех групп инсталляций: клиент, тип транспорта, статус, количество единиц. Фильтрация по статусу и компании. Отправная точка для любого сценария менеджера.

02
Детали установки и отдела

Из списка — в конкретную инсталляцию: назначенные установщики, статус по каждому ТС, дата, отдел-заказчик. Отдельный вид по отделу собирает все его установки в одном месте.

03
Реестр установщиков

Список всех монтажников с поиском: имя, статус, количество активных заданий, компания. Кликабельная строка — переход в профиль конкретного установщика.

04
Профиль и создание установщика

Профиль — вся история и текущая нагрузка установщика: назначенные ТС, группы, контакты. Форма создания нового идентична по структуре профилю — один и тот же мыслительный паттерн для чтения и записи.

05
Транспортные средства

Список всех ТС с типом, номером, статусом установки и назначенным установщиком. Разные типы транспорта — единый формат карточки; визуальная иконка типа не даёт перепутать при больших списках.

Единый портал вместо трёх разрозненных таблиц

01 Менеджер видит все группы установок в одном месте и фильтрует по статусу и клиенту
02 Из любой группы — в детали конкретной инсталляции и отдела за один клик, без потери контекста
03 Реестр установщиков с поиском; профиль и форма создания используют одну и ту же структуру
04 Транспортные средства разных типов — в едином формате с визуальным различением по иконке типа
05 Три модуля связаны сквозными переходами: установка → установщик → транспорт — в любую сторону
Рефлексия

Что вынес

Когда сущности связаны между собой, важно сначала нарисовать схему переходов, а не сразу экраны. Иначе получаешь пять отдельных страниц вместо единого продукта — и переделываешь навигацию в конце.

Форма создания и профиль должны использовать одинаковую структуру полей — пользователь, который читает профиль, должен без обучения понимать, как заполнять форму. Это снижает когнитивную нагрузку и ошибки при вводе.
